## What 30 Years Gets You

Most AI Engineer candidates are strong on the new parts — LLM APIs, vector stores, agent frameworks. The gaps tend to show up in the engineering fundamentals: system design under load, failure modes, eval discipline, knowing when *not* to use an LLM.

Three decades of shipping production software changes how you approach those problems. I'm not faster at prompt engineering. I'm better at building systems that are reliable, observable, and maintainable when the model does something unexpected.

That's the bet.
